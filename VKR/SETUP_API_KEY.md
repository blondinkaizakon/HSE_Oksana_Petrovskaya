# Настройка API ключа Gemini

## Проблема
Приложение не отвечает, потому что не настроен API ключ для Gemini.

## Решение

1. **Получите API ключ Gemini:**
   - Перейдите на https://ai.google.dev/
   - Войдите в аккаунт Google
   - Создайте новый API ключ или используйте существующий

2. **Добавьте API ключ в файл `.env.local`:**

   ```bash
   ssh root@92.53.104.183
   cd /root/blondinkaizakon
   nano .env.local
   ```

   Замените `your_api_key_here` на ваш реальный API ключ:
   ```
   VITE_GEMINI_API_KEY=ваш_реальный_api_ключ_здесь
   GEMINI_API_KEY=ваш_реальный_api_ключ_здесь
   ```

   Сохраните файл (Ctrl+X, затем Y, затем Enter)

3. **Перезапустите приложение:**

   ```bash
   # Остановить текущий процесс
   pkill -f "blondinkaizakon.*vite"
   
   # Запустить заново
   cd /root/blondinkaizakon
   npm run dev > dev.log 2>&1 &
   ```

4. **Проверьте работу:**

   - Откройте в браузере: http://92.53.104.183:3000
   - Откройте консоль разработчика (F12)
   - Попробуйте отправить сообщение или загрузить файл
   - Проверьте, нет ли ошибок в консоли

## Альтернативный способ (через терминал на сервере)

```bash
cd /root/blondinkaizakon
cat > .env.local << 'EOF'
VITE_GEMINI_API_KEY=ваш_реальный_api_ключ_здесь
GEMINI_API_KEY=ваш_реальный_api_ключ_здесь
EOF
```

**ВАЖНО:** Замените `ваш_реальный_api_ключ_здесь` на ваш настоящий API ключ!

## Проверка

После настройки API ключа в консоли браузера (F12) не должно быть ошибок вида:
- "API key not found"
- "authentication failed"
- "GEMINI_API_KEY не настроен"

Приложение должно начать отвечать на запросы и анализировать файлы.
